# qqDecoder

这是一个基于 GitHub Actions 实现的、自动监控并反编译腾讯 QQ 安卓版的工作流项目。

通过定时任务或手动触发，工作流会自动获取官方最新的 APK 直链。在虚拟机环境中配置 Swap 虚拟内存和大内存堆后，使用 JADX 将所有的 DEX 文件还原为 Java 源码，并自动发布到 GitHub Releases 中供下载。

## 工作原理

1. **版本检测**
   工作流每天定时运行，通过抓取 Telegram 频道的免登录 Web 预览页获取最新的 QQ 官方下载直链。

2. **对比与去重**
   检测到的最新直链会与仓库中的 `latest_version.txt` 进行比对。只有当链接发生变化（即检测到新版本）或手动触发时，才会执行后续的反编译流程，以此避免重复运行并节省 Actions 额度。

3. **磁盘与内存优化**
   由于 QQ 的代码量极其庞大，GitHub 虚拟机的默认配置极易因为内存和磁盘溢出而崩溃。工作流在运行初期会清理虚拟机内的无用组件，挂载 18GB Swap 虚拟内存，并分配 20GB 的 JVM 堆内存以确保 JADX 稳定运行。

4. **反编译与容错机制**
   使用 JADX 反编译提取出的所有 `classes*.dex` 文件。针对大项目可能出现的局部混淆或损坏类（导致 JADX 返回退出码 3），工作流提供了容错兼容处理，保证能顺利完成并导出绝大部分源码。

5. **自动发布**
   反编译完成后，自动将 Java 源码压缩，使用 GitHub CLI 工具自动创建对应的 Release，上传 ZIP 产物并附带简单的版本说明。

## 配置与部署

如果你希望在自己的仓库中运行此项目，请按照以下步骤进行配置：

### 1. 开启工作流写权限
由于工作流需要自动更新 `latest_version.txt` 记录并发布 Releases，必须为 GITHUB_TOKEN 开启写入权限：
1. 打开您的 GitHub 仓库，点击顶部的 **Settings**。
2. 在左侧菜单中找到 **Actions** -> **General**。
3. 滚动到页面底部，找到 **Workflow permissions**，将其修改为 **Read and write permissions** 并点击保存。

### 2. 运行工作流
* **定时自动运行**：默认配置了每日定时触发器（北京时间早上 8:00）。它会自动执行检测，只有在有新版本发布时，才会跑完整个反编译和发布流程。
* **手动强制运行**：你可以在仓库的 **Actions** 页面，选择 **Auto Decompile QQ APK** 工作流，点击 **Run workflow** 手动触发。
  * **decompile_all**：手动运行时是否进行完整反编译。若为 `false`，则仅导出 `com.tencent.mobileqq` 和 `com.tencent.qqnt`。
  * **decompile_mobileqq / decompile_qqnt**：手动局部反编译时，控制是否包含这两个核心包。

## 性能与线程微调

在每日自动全量反编译时，默认使用双线程（`--threads-count 2`）以加快编译速度。

如果后续因为 QQ 版本的体积进一步膨胀，导致虚拟机在反编译阶段因内存耗尽（OOM）而中断，可以编辑工作流 YML：
* 找到 JADX 完整反编译命令处的 `--threads-count 2`，将其修改为 `--threads-count 1`。单线程模式的处理时间会更长，但能显著降低瞬时的内存峰值。
